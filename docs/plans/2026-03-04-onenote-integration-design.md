# OneNote Integration Design

**Date:** 2026-03-04
**Status:** Implemented

## Overview

Adds a "Send to OneNote" button to `MemoViewer` that authenticates with Microsoft via MSAL.js (OAuth2 PKCE) and creates a OneNote page via MS Graph API. Implemented entirely in the frontend ‚Äî no backend changes required.

## Architecture

```
MemoViewer
  ‚îî‚îÄ "Send to OneNote" button
       ‚îú‚îÄ useMicrosoftAuth (MSAL.js loginPopup / acquireTokenSilent)
       ‚îî‚îÄ OneNotePickerModal
            ‚îî‚îÄ useOneNote (MS Graph API)
                 ‚îú‚îÄ GET /me/onenote/notebooks
                 ‚îú‚îÄ GET /me/onenote/notebooks/{id}/sections
                 ‚îî‚îÄ POST /me/onenote/sections/{id}/pages
```

## Auth Flow

1. User clicks "Send to OneNote"
2. If no MS account active ‚Üí `loginPopup()` with scopes `['Notes.ReadWrite', 'User.Read']`
3. `OneNotePickerModal` opens and fetches notebooks via `acquireTokenSilent`
4. User selects notebook ‚Üí sections are fetched
5. User selects section ‚Üí `POST /me/onenote/sections/{id}/pages` with HTML body
6. Success state shown; modal closes on "Done"

Token acquisition falls back to `acquireTokenPopup` when silent refresh fails (e.g., consent required).

## Configuration

Add to `web/.env`:
```
VITE_MSAL_CLIENT_ID=<azure-ad-app-client-id>
```

Azure AD app requirements:
- Supported account types: personal + organizational (common endpoint)
- Redirect URI: SPA type ‚Üí `https://<your-netlify-domain>` + `http://localhost:5173`
- API permissions: Microsoft Graph ‚Üí Delegated ‚Üí `Notes.ReadWrite`

## Decisions

- **Frontend-only**: MS Graph supports direct browser calls with PKCE ‚Äî no proxy needed
- **sessionStorage cache**: Tokens not persisted across tabs; acceptable for this use case
- **`/common` authority**: Supports both personal Microsoft accounts and organizational accounts
- **loginPopup over redirect**: Avoids full-page navigation, better UX in SPA
